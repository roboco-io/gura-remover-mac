# Contributing

`gura-remover-mac`에 기여하기 전에 아래 기준을 먼저 확인해 주세요. 이 프로젝트는 macOS에서 보안 모듈 흔적을 탐지하고 제거하는 CLI이므로, 일반 유틸리티보다 보수적인 변경 기준을 적용합니다.

## 기본 원칙

- 파괴적 변경보다 안전한 기본값을 우선합니다.
- 삭제 로직을 바꿀 때는 반드시 `--dry-run`, 백업, 복구 흐름을 함께 검토합니다.
- 새 의존성 추가는 꼭 필요한 경우에만 진행합니다.
- 작은 PR을 선호합니다. 문서, 탐지, 제거 로직을 한 번에 크게 섞지 않는 편이 좋습니다.

## 개발 환경

- macOS 13 이상
- Swift 6.2
- Xcode Command Line Tools

## 로컬 작업

저장소를 클론한 뒤 아래 명령으로 기본 검증이 가능합니다.

```bash
make build
make test
make scan ARGS='--json'
make doctor ARGS='--json'
```

직접 실행이 필요하면:

```bash
swift run gura --help
swift run gura help remove
```

## 변경 가이드

### 탐지 시그니처 변경

- 가능하면 `Sources/GuraCore/Resources/signatures/default.json`만으로 해결 가능한지 먼저 확인합니다.
- 특정 벤더/제품군을 추가할 때는 오탐 가능성과 근거 경로를 함께 설명해 주세요.
- 실제 시스템 경로, `launchd`, `pkg receipt`, 시스템 확장 여부를 구분해서 다루는 편이 좋습니다.

### 제거 로직 변경

- 파일 삭제 외에 프로세스 종료, `launchd` 해제, receipt 정리, 시스템 확장 제거 시도에 미치는 영향을 함께 설명해 주세요.
- 고위험 항목 판정이 바뀌면 이유를 PR 설명에 남겨 주세요.
- 가능하면 테스트로 행동을 고정한 뒤 수정해 주세요.

### 문서 변경

- README, PRD, 기여 가이드 간의 용어를 맞춰 주세요.
- CLI 명령 예시는 실제 현재 구현과 일치해야 합니다.

## 테스트 기대사항

코드 변경 시 최소한 아래 중 해당 범위를 확인해 주세요.

- `make test`
- `swift build -c release --product gura`
- 필요한 경우 `gura scan`, `gura doctor`, `gura remove --dry-run` 수동 확인

문서 전용 변경이면 별도 빌드가 꼭 필요하지는 않지만, 명령 예시와 경로는 현재 저장소 기준으로 다시 확인해 주세요.

## 이슈와 PR 작성

- 버그 리포트에는 macOS 버전, CPU 아키텍처, 실행한 명령, 기대 결과, 실제 결과를 포함해 주세요.
- 제거 실패나 오탐 이슈에는 민감한 정보가 없는 범위에서 로그/출력 일부를 함께 남겨 주세요.
- PR에는 변경 요약, 영향 범위, 테스트 결과, 남은 리스크를 적어 주세요.

## 보안 및 안전

- 실제 사용자 시스템 경로, 금융/공공 사이트 흔적, 백업 데이터가 포함된 자료를 올릴 때는 민감 정보를 먼저 제거해 주세요.
- 삭제 로직이 사용자 데이터나 시스템 기본 구성에 영향을 줄 수 있다면 이슈 또는 PR에서 명확히 경고해 주세요.
