# 구라 제거기 for MAC

## 1. 제품 개요

- 제품명(가칭): GuraCleaner for Mac  
- 목적: 인터넷 뱅킹·공공기관 이용 시 설치되는 각종 보안·키보드 보안·광고성 유틸리티를 탐지·삭제해 맥 환경을 **경량화**하는 macOS용 CLI 도구.  
- 대상 플랫폼: macOS (Apple Silicon / Intel, 최소 지원 OS 버전은 추후 확정).  

***

## 2. 타깃 사용자 및 페르소나

- 주 사용자:
  - 한국 인터넷 뱅킹·공공 사이트를 자주 사용하는 맥 사용자.
  - 시스템에 뭐가 설치됐는지 알고 싶어 하는 개발자/파워유저.
  - GUI보다 터미널 자동화와 스크립트 연동을 선호하는 사용자.
- 대표 페르소나:
  - “맥북으로 회사 업무·개인 뱅킹을 같이 보는 30대 직장인, 은행/관공서용 프로그램이 쌓여서 정리하고 싶은 사용자.”
  - “테스트용으로 각종 보안 모듈을 설치해보는 개발자, 한 번에 싹 지우고 초기 상태로 돌리고 싶은 사용자.”

***

## 3. 문제 정의

- 문제 상황:
  - 인터넷 뱅킹·공공 사이트 이용 과정에서 여러 에이전트/보안 모듈/런처가 설치되고, 삭제 경로가 불명확하거나 복잡하다.
  - 일부는 로그인/결제 후에도 백그라운드 프로세스나 LaunchAgent, kext, 서비스 형태로 남아 리소스를 사용한다.
- 사용자가 겪는 Pain Point:
  - “뭘 설치했는지 목록을 한눈에 볼 수 없다.”
  - “삭제가 번거롭고, 수동으로 찾기 어렵다.”
  - “실수로 꼭 필요한 모듈을 지우는 게 불안하다.”

***

## 4. 목표 및 비목표

- 목표(Do):
  - 대표적인 뱅킹/공공 사이트 관련 맥용 보안 모듈 및 유사 블로트웨어를 자동 탐지하고 안전하게 제거.
  - 어떤 항목이 왜 블로트웨어 후보인지 터미널 출력과 로그로 명확히 보여주기.
  - 삭제 전 시스템 복원 가능성(예: 설정 백업, 삭제 로그)을 제공.
- 비목표(Don’t):
  - 악성코드/백신 수준의 실시간 보안 솔루션 제공.
  - 윈도우용 구라제거기와 1:1 동작/리스트 완전 동일화.
  - 루트킷/커널 레벨 포렌식 도구 수준의 기능 제공.

***

## 5. 주요 기능 요구사항

### 5.1 스캔 및 탐지

- 기능:
  - 알려진 뱅킹/공공 사이트 관련 앱, 런처, 에이전트, 확장 프로그램을 탐지.
  - 탐지 대상 예:
    - `/Applications`, `~/Applications`의 앱 번들.
    - `~/Library/LaunchAgents`, `/Library/LaunchAgents`, `/Library/LaunchDaemons`의 런치 플리스트.
    - `~/Library/Internet Plug-Ins`, 브라우저 확장(가능한 범위).
    - 시스템 환경설정에 등록된 로그인 항목.
  - 각 항목별로 다음 정보를 출력:
    - 이름, 벤더/번들 ID, 설치 위치.
    - 예상 역할(예: “키보드 보안 모듈”, “공공 사이트용 실행 런처”).
    - 삭제 난이도/리스크 레벨(저/중/고).  
- 초기 탐지 우선순위:
  - 1차 우선 대상:
    - `TouchEn nxKey` 계열.
    - `WizIN-VeraPort` 계열.
    - `INISAFE CrossWeb EX` 계열.
  - 2차 유력 대상:
    - `TouchEn Transkey`, `TouchEn nxFirewall`, `TouchEn nxWeb`, `Key# Biz`.
  - 3차 검증 필요 대상:
    - `AnySign/AnySign4PC`, `CrossCert/CrossEX` 등.
  - 각 시그니처의 근거와 신뢰도는 `docs/stack-and-target-research.md`에 따로 기록하고, 코드에는 `confidence` 필드를 포함한다.

### 5.2 제거(언인스톨)

- 기능:
  - 선택된 항목을 한 번에 제거(일괄 삭제) 혹은 개별 삭제.
  - 비대화형 모드와 인터랙티브 확인 모드를 모두 지원.
  - 삭제 수행 항목:
    - 앱 번들 삭제.
    - 관련 LaunchAgent/Daemon 플리스트 제거.
    - 관련 로그/캐시/설정 파일(옵션) 제거.
  - 삭제 전/후 처리:
    - 삭제 전 권한 상승(필요 시 admin 권한 요청).
    - 삭제 로그 기록(파일 경로, 시간, 수행 결과).
    - 삭제 실패 시 상세 오류 메시지 제공.  

### 5.3 세이프 모드 / 화이트리스트

- 기능:
  - `--safe` 플래그 기반 “안전 모드”:
    - 기본값으로는 **저·중 리스크 항목만** 삭제 대상으로 허용.
    - 시스템에 큰 영향을 줄 수 있는 항목(고 리스크)은 별도 확인 인자(예: `--force-high-risk`) 필요.
  - 화이트리스트 관리:
    - 사용자가 “항상 무시”로 설정한 항목은 다음 스캔에서 제외.

### 5.4 프로파일 / 프리셋

- 기능:
  - “은행 관련만 보기”, “공공 사이트 관련만 보기”, “전체 보기” 등 필터 프리셋을 서브커맨드/플래그로 제공.
  - 추후 업데이트로 특정 은행/기관별 프로파일(예: “OO은행 세트”, “홈택스 세트”) 제공을 고려.

### 5.5 로그 및 리포트

- 기능:
  - 스캔/삭제 이력을 날짜별로 기록.
  - 간단한 텍스트/JSON 로그 파일로 내보내기(디버깅, 이슈 리포트용).
  - `history` 또는 `log` 계열 명령으로 최근 작업 내역 확인 가능.

## 6. 비기능 요구사항

- 성능:
  - 전체 스캔 시간: 일반적인 사용자 환경 기준 5초 내 목표(최대 10초).
  - 백그라운드 CPU 점유율 최소화.
- 안정성:
  - 삭제 실패 시 시스템 부팅/로그인을 방해하지 않도록 방어적 구현.
  - 일부 항목 삭제 후 재부팅이 필요한 경우 명확한 안내.
- 보안/프라이버시:
  - 네트워크 통신 최소화, 기본값 오프라인 동작.
  - 로그에 민감한 정보(계정명, 토큰 등)를 남기지 않기.
- 호환성:
  - Apple Silicon / Intel 모두 지원.
  - 최소 지원 macOS 버전(ex: 12 Monterey 혹은 13 Ventura 이상) 명시.  

***

## 7. CLI UX 요구사항

- UX 방향:
  - “터미널에서 빠르게 스캔하고, 출력 결과를 보고, 필요한 항목만 안전하게 지운다”는 흐름으로 단순화.
- 핵심 커맨드:
  - `scan`
    - 전체 스캔 실행.
    - 탐지된 항목 수, 위험도 요약, 카테고리별 집계를 출력.
  - `list`
    - 테이블 형태 또는 `--json` 형태로 이름, 역할(설명), 위치, 위험도 출력.
    - 필터 옵션(은행/공공/전체, 위험도별, 벤더별) 지원.
  - `remove`
    - 삭제 대상 항목 요약 출력.
    - 기본값은 확인 프롬프트를 표시하고, 자동화용 `--yes` 옵션 제공.
  - `restore` 또는 `rollback`
    - 가능할 경우 최근 삭제 작업 복구 또는 백업 기반 복원 지원.
- 출력 원칙:
  - 사람이 읽기 쉬운 기본 포맷과 자동화 친화적인 JSON 포맷을 모두 제공.
  - 오류 메시지는 원인, 실패 경로, 권한 필요 여부를 명확히 포함.
- 언어:
  - 기본: 한국어.
  - 옵션: 영어(국제 기여자/개발자용).

## 8. 기술 스택 및 구조(확정안)

- 앱 타입:
  - 네이티브 macOS CLI 실행 파일로 확정.
- 언어 및 런타임:
  - `Swift 6` + `Swift Package Manager`.
  - 최소 지원 OS는 `macOS 13 Ventura` 이상으로 설정.
- CLI 프레임워크:
  - `apple/swift-argument-parser` 사용.
  - 기본 커맨드 구조는 `scan`, `list`, `remove`, `restore`, `history`, `doctor`.
- 표준 라이브러리/프레임워크:
  - `Foundation`: 파일 시스템, 프로세스 실행, JSON/Property List 처리.
  - `OSLog`: 구조화 로그.
  - Apple 시스템 명령(`launchctl`, `pkgutil`, `systemextensionsctl`, `log`, `codesign`)은 `Process` 래퍼로 호출.
- 핵심 모듈:
  - 스캐너 모듈: 파일 시스템/런치 에이전트/로그인 아이템 탐색.
  - 룰 엔진: “구라 후보”를 식별하는 시그니처 DB(번들 ID, 이름 패턴, 경로 패턴 등).
  - 제거 모듈: 삭제/권한 요청/롤백 일부 지원.
  - 로깅 모듈: JSON/텍스트 기반 로그 관리.
  - CLI 인터페이스 모듈: 서브커맨드, 플래그, 인터랙티브 프롬프트 처리.
- 시그니처 저장 방식:
  - 버전 관리 가능한 JSON 파일로 유지.
  - 항목 예시 필드: `id`, `vendor`, `product`, `bundleIds`, `paths`, `launchdLabels`, `pkgReceipts`, `risk`, `confidence`.
- 권한 전략:
  - v1은 별도 privileged helper를 두지 않고, 관리자 권한이 필요한 명령은 `sudo` 전제로 실행.
  - 시스템 확장/kext/daemon 제거 시 재부팅 필요 여부를 결과에 명시.
- 테스트 전략:
  - 단위 테스트는 `Swift Testing`.
  - 통합 테스트는 `swift test` 기반 fixture 디렉터리와 샘플 plist/receipt/로그를 사용.
- 포맷/품질:
  - 코드 포맷은 `swift-format`.
  - 린트는 초기에는 포맷 중심으로 시작하고, 필요 시 `SwiftLint`를 추가 검토.
- 시그니처 업데이트:
  - 로컬에 기본 시그니처 포함.
  - 추후 GitHub Release나 원격 JSON을 통한 시그니처 업데이트 고려.
